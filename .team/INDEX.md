## 📝 Document Maintenance

### How Documents Are Updated

- **Daily**: [Progress Tracking](projects/PROGRESS.md) - coordinator updates
- **Weekly**: [Project Schedule](projects/SCHEDULE.md) - plan for upcoming week
- **Monthly**: [Project Roadmap](projects/ROADMAP.md) - quarterly review
- **As needed**: Role guides and other reference docs

### How to Request Updates

1. Check if document is current
2. Identify what needs updating
3. Notify Coordinator and Writer
4. Create issue or pull request
5. Review and merge

### Version Control

All team documents are version controlled in Git:
- `.team/` directory contains all docs
- Changes tracked with git history
- Updates reviewed before merge
- Archive old versions in `archive/` folder
