Experiment setup

End-to-end workflow grouped by stage

  1. A

    Data acquisition & normalization

    Load/archive posts, coerce shape, normalize text, preserve metadata (images, references, board/site, tripcode), and apply optional admin-drop exclusion.

  2. B

    Representation & clustering

    Encode drops with sentence embeddings, reduce dimensions with UMAP, cluster with HDBSCAN/BERTopic (or TF-IDF + KMeans baseline), and produce per-topic labels from c-TF-IDF.

  3. C

    Validity & robustness checks

    Score each run with silhouette and c_v coherence, then run parameter sweeps (model × min cluster size) and compute topic persistence by Jaccard overlap.

  4. D

    Output generation & interpretation

    Export JSON/CSV payloads, render figures in `drops.html`, and compare computed findings against peer-reviewed conclusions with explicit scope boundaries.
