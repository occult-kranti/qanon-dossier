The Drops, Quantified

Every Q drop turned into a vector, then clustered into topics that organise themselves — mapped, counted, and made searchable. Read here as discourse objects: the point is to measure the drops (their themes, repetition, and vagueness), not to amplify them.

This page reads the output of the clustering pipeline (qdrops_clustered.json). The underlying corpus is the research-only JSON-QAnon archive — all known drops through November 2022 — whose own author prefaces it: "QAnon is a dangerous cult… for research purposes only." The analytical value of clustering is exactly that it makes the drops legible as data: it shows how a handful of themes recur across thousands of posts, and how much of the corpus is too generic to cluster at all — a quantitative footprint of the vagueness that lets believers retrofit meaning onto almost anything.

Stability across runs — which topics are real?

Clustering is parameter-sensitive; trust the topics that persist

A single clustering can mislead. Run the corpus several ways (different embedding models and min_cluster_size values) and a topic that keeps reappearing is probably real; one that shows up under only one setting is likely an artifact. Data & method — reproduce it

Download the dataset, run the pipeline, refresh this page

Why HDBSCAN, not k-NNMethod note

k-NN is a supervised classifier — it needs labels and so cannot discover topics on its own. To let clusters form themselves on short, cryptic text, the pipeline embeds each drop, reduces dimensionality with UMAP, and clusters with HDBSCAN (density-based: it chooses the number of topics itself and sets genuine outliers aside as "noise"). BERTopic wraps this and labels topics with class-based TF-IDF. The legitimate way k-NN enters clustering is as a graph — a k-nearest-neighbour graph fed to Leiden/Louvain community detection — which the code notes as an alternative; a plain TF-IDF + KMeans baseline is also included.

  1. 1

    Get the dataset & the pipeline

    The script downloads the corpus automatically, or grab it yourself:

    pip install -r requirements.txt python qdrops_cluster.py --inspect # sanity-check the data shape (optional) python qdrops_cluster.py # recommended: BERTopic (auto-downloads posts.json)
    JSON-QAnon (GitHub) Kaggle mirror DOI
  2. 2

    Pick a clustering technique

    python qdrops_cluster.py --method bertopic --min-cluster-size 15 # default, self-forming topics python qdrops_cluster.py --method hdbscan # same, without BERTopic python qdrops_cluster.py --method kmeans -k 14 # quick baseline, no torch

    Smaller --min-cluster-size = more, finer topics (and a smaller "noise" bucket). Output: qdrops_clustered.json + a CSV summary.

  3. 3

    View it here

    Drop qdrops_clustered.json into the same folder as this page (or use the Load a JSON file button up top) and reload. Use Unload in the status bar to clear it and load a different file.

Citation & ethicsResearch only

Cite the dataset as Kingsman, J. (2025), JSON-QAnon, DOI 10.13140/RG.2.2.28778.32964. The drops are reproduced here as primary-source artifacts for analysis only; this dossier treats them critically throughout (see the Critical Review) and does not endorse their content. Images referenced by the drops are deliberately excluded.

What the clustering shows — and doesn't. Topic clusters reveal recurring themes and the share of the corpus that resists grouping; they do not measure truth, intent, or authorship. Cluster boundaries shift with parameters (embedding model, min_cluster_size, UMAP settings), so treat topic counts as a structured reading of the corpus rather than a fixed fact — exactly the calibration argued on the Critical Review page.
